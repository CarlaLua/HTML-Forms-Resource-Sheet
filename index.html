HTML Forms Reference sheet

HTML Tags

Tag Name Description
<form> Form The wrapper element for all HTML forms.
<input> Input The most used form element. It always has a type attribute.
<label> Label The <label> element defines a label for several form elements.
<select> Select The <select> element defines a drop-down list
<option> Option The <option> element defines an option that can be selected. By default, the first item in the drop-down list is selected. To define a pre-selected option, add the selected attribute to the option
<button> Button This element defines a clickable button
<textarea> Textarea The <textarea> element defines a multi-line input field (a text area)
<fieldset> Fieldset This element is used to group related data in a form
<legend> Legend This element defines a caption for the <fieldset> element
<datalist> Datalist This element specifies a list of pre-defined options for an <input> element. Users will see a drop-down list of the pre-defined options as they input data. The list attribute of the <input> element, must refer to the id attribute of the <datalist> element.
<output> Output This element represents the result of a calculation (like one performed by a script)
<optgroup> Optgroup This tag is used to group related options in a <select> element (drop-down list)

HTML Attributes

Attribute Name Description
type Type The type attribute belongs to the <input> element. The values are text, radio, checkbox, submit or button.
id ID An attribute of the <input> tag, it should match the for attribute of the <label> element to bind them together.
name Name each input field must have a name attribute to be submitted. If the name attribute is omitted, the value of the input field will not be sent at all.
value Value The value attribute specifies the value of an <input> element.
for For An attribute of the <label> tag, it should match the id attribute of the <input> element to bind them together.
action Action Specifies where to send the form-data when a form is submitted
target Target Specifies where to display the response that is received after submitting the form
method Method Specifies the HTTP method to use when sending form-data
accept-charset Accept-Charset Specifies the character encodings used for form submission
autocomplete Autocomplete Specifies whether a form should have autocomplete on or off
enctype Enctype Specifies how the form-data should be encoded when submitting it to the server (only for method="post")
novalidate Novalidate Specifies that the form should not be validated when submitted
rel Rel Specifies the relationship between a linked resource and the current document
